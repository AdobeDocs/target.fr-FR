# Indicateurs d’accès dans Adobe Target {#log-in}

Pour commencer à utiliser les indicateurs, demandez l’accès, puis ouvrez les indicateurs dans Adobe Target.

## Demander l’accès {#request-access}

Demandez l’accès par le biais du système de gestion des accès de votre entreprise. Consultez [Demande d’accès](request-access.md) pour obtenir des instructions détaillées.

Une fois votre demande approuvée, vous recevrez les autorisations associées au rôle accordé par votre administrateur.

## Indicateurs d’accès {#log-in-steps}

Une fois l’accès accordé :

1. Accédez à [&#128279;](https://experience.adobe.com/) et connectez-vous avec les informations d’identification de votre organisation.
1. Ouvrez **&#x200B;**.
1. Sélectionnez **Indicateurs** dans la navigation de Target.
1. Sélectionnez la sandbox appropriée. Voir [&#x200B; Présentation des environnements](environments-overview.md) pour plus d’informations.

## Premières étapes après l’accès aux indicateurs {#first-steps}

Après avoir ouvert Indicateurs, vérifiez que votre application est répertoriée. Les applications sont gérées par les administrateurs. Si votre application n’est pas répertoriée, contactez votre administrateur pour l’ajouter. Pour plus d’informations, consultez [Gestion des applications](../applications/manage-applications.md).

## Terminologie clé {#terminology}

| Terme | Définition |
| --- | --- |
| **Application** | Application à contrôler à l’aide d’indicateurs de fonctionnalité. |
| **Indicateur de fonctionnalité/Groupe de fonctionnalités** | Les artefacts créés dans Indicateurs pour les tests de fonctionnalités et les déploiements contrôlés. |
