# Demander l’accès {#request-access}

L’accès aux indicateurs est géré au niveau de l’équipe. Pour utiliser les indicateurs, vous devez être ajouté à une équipe avec le rôle approprié.

## Conditions préalables {#prerequisites}

Avant de demander l’accès, vérifiez les points suivants :

* Vous savez quelle équipe rejoindre. Vérifiez auprès de votre chef de produit ou d’ingénierie que votre équipe et son administrateur/administratrice sont identifiés.
* Vous savez à quel sandbox vous devez accéder.

## Demander l’accès {#request-steps}

Contactez votre représentant Adobe pour demander l’accès.

## Rôles disponibles {#roles}

Le rôle qui vous est affecté détermine les actions que vous pouvez effectuer dans la console. Contactez votre administrateur si vous ne savez pas quel rôle demander.

| Rôle | Description |
| --- | --- |
| **Admin** | Accès complet à la console. Peut gérer des applications. |
| **Propriétaire de la version de produit** | Possibilité de créer et de gérer des indicateurs de fonctionnalités et des groupes de fonctionnalités, et de publier des fonctionnalités pour des utilisateurs externes. |

## Une fois l’accès accordé {#after-access}

Une fois votre demande approuvée, ouvrez Indicateurs dans Adobe Target et vérifiez que vos applications sont visibles. Pour connaître les étapes suivantes, voir [Indicateurs d’accès dans Adobe Target](log-in-to-the-console.md).

## Besoin d&#39;aide ? {#need-help}

Si vous ne savez pas quelle équipe rejoindre, ou s’il n’existe pas encore d’équipe pour votre projet, contactez l’administrateur Flags de votre organisation.
